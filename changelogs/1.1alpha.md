# Commands-Collection v1.1alpha

Welcome to the first alpha release of Commands-Collection!

(I do wonder why 1.1 though, yes I'm the maker of this project)

For short, call Commands-Collection CC. And we will use that name from now on.

## Features

### Eye-candy:3

(not really)

Colors are now used widely in CC. For logging. For `--help`. For errors. For informations.

`dir` now lets you create different arrangement of columns, different string/character for each file type. Presets for CC's own layout, GNU's `ls` and Windows's `dir` are still present and can be set using `DIR_PRESET` environment variable. Sadly it doesn't color names right now, as it's not easy to implement.

### Localizations

It's finally coming. And the best? No *dependencies* (for runtime).

CC supports localizations from whatever path, by modifying `CC_I18N_LOC` environment variable.

For you maintainers, `gettext` is required to generate files. Installing is easy: either via your OS's package manager OR XMake's XRepo:

```bash
$ xrepo install gettext
```

Targets for generating `.po` and `.mo` files have been added to `xmake.lua`.

### Build system changes

Make is no longer used, along with Makefile.fpc. Instead, we now use this not-so-new thing: [XMake](https://xmake.io).

It uses Lua. It runs fine on many platforms. It *let* you cross-compile (I wonder if it supports FPC though). It supports FPC!

`fpc.cfg` was added in order to make it easier to compile any program without copy-pasting required flags. All you need is the location of FPC's `fpc.cfg`, as uhm, the compiler asks you to:

```bash
lebaothecook@ubuntu:/mnt/DataDisk/Projects/Commands-collection$ fpc src/cc.base.pp
Hint: End of reading config file fpc.cfg
(lines of output)
Fatal: Can't find unit system used by cc.base
Fatal: Compilation aborted
Error: /usr/bin/ppcx64 returned an error exitcode
```

To fix this one, pass @`<where to that fpc.cfg>` to the compiler. On Linux, it should be put in /etc.

You don't need to that with XMake - unless you want or have to use another path: use this one:

```bash
$ xmake f --fpc_conf=<path>
$ xmake # Build as usual
```

You can tell XMake to append a prefix to each program names:

```bash
$ xmake f --output-prefix=<whatever>
$ xmake
```

To build a program / target:

```bash
$ xmake --help # Check its output's end for targets
$ xmake b <target>
```
